# DESIGN.md - 南湖调度系统（Nanhu Scheduling System）

## 1. Visual theme and atmosphere

南湖调度系统作为面向企业级的工业级GPU资源调度、K8s状态监控及任务迭代管理平台，核心设计理念围绕「专业克制、高效可靠、理性精准」展开。界面整体氛围追求低干扰、高信息密度的工具属性表达，而非装饰性视觉呈现——这是因为系统的核心用户为运维工程师、研发人员及管理员，其核心诉求是快速获取调度状态、精准执行操作、高效定位问题，而非视觉愉悦性。

设计上通过清晰的信息层级、标准化的组件规范、克制的色彩运用，弱化视觉干扰，强化数据与操作的核心地位；同时结合「区域化主题（zone）」适配不同部署环境的视觉识别需求，既保证全局视觉一致性，又满足多区域运维的场景化区分，最终实现「工具级的实用性」与「企业级的专业性」平衡，让用户聚焦于调度、监控核心业务，而非界面本身。

## 4. Components

### 4.1 按钮与交互 (Action & Interaction)

*   **基础样式**：主按钮背景采用 `#0D8356` 纯色，圆角 6px，去除了多余的阴影以保持扁平化。
    
*   **定制规则**：当按钮处于 Disabled 状态时，背景色固定为 `rgba(0,0,0,0.25)`。
    

### 4.2 监控图表 (Monitoring Charts)

*   **样式规范**：图表背景透明，网格线极淡。曲线颜色根据指标区分（GPU-紫、CPU-蓝、内存-黄）。
    
*   **交互逻辑**：Hover 触发 Tooltip 浮窗，支持实时采样点的高亮显示。
    

### 4.3 数据表格与日志 (Data & Logs)

*   **表格**：极简边框，支持 `.highlight` 过滤状态高亮。针对长路径字段使用末尾截断，并配备一键复制按钮（`CopyButton`）。
    
*   **日志查看器**：Terminal 样式，等宽字体，支持自动滚动与关键词高亮（如红色标记 "Error"）。
    

### 4.4 布局组件 (Navigation & Sidebar)

*   **LeftArea**：浅色底，选中态加粗并变为主色，支持多级树状结构折叠。
    
*   **ConfigDialog**：用于复杂资源配置，层级最高，背景带 Level 3 阴影。

## 5. Layout

布局严格遵循 B 端企业级效率工具的规范：

*   **原子单位**：以 **4px** 为基础间距，严格遵循 **8px** 网格系统（8/16/24/32px）。
    

*   **布局结构**：
    

**Top Bar**：固定高度 56px，承载集群切换与全局操作。

**Sidebar (LeftArea)**：固定宽度，支持收缩为图标模式。

**MainContent**：24px 内边距，内部卡片模块间距 16px。

*   **圆角体系**：徽章 2px / 控件 4px / 按钮 6px / 容器卡片 8px。

## 6. Depth and elevation

调度系统为典型的数据密集型操作平台，阴影层级被克制使用，主要依靠 Z 轴区分模态交互：

| Level | Usage Scenario | Shadow Value (RGBA) |
| --- | --- | --- |
| Level 0 | 页面背景、侧边栏、静态列表 | `none` / 无阴影，纯平贴合 |
| Level 1 | 详情页内的功能卡片、面板 | `0 1px 2px rgba(0,0,0,0.1)` / 极其轻微的隔离感 |
| Level 2 | 下拉菜单、环境切换浮窗、Tooltip | `0 4px 12px rgba(0,0,0,0.12)` / 临时性交互层 |
| Level 3 | 任务配置弹窗、全局通知 | `0 8px 24px rgba(0,0,0,0.16)` / 强制打断操作层级 |

## 7. Do's and don'ts

### Do

*   **数据对齐**：所有监控指标的 X 轴（时间轴）必须垂直对齐，确保多指标联动观察的准确性。
    
*   **强化工具属性**：所有系统路径、ID 必须配备一键复制（`CopyButton`）。
    
*   **安全合规**：所有操作界面必须覆盖用户昵称水印（透明度 0.1，角度 -45°）。
    

### Don't

*   **禁止装饰过载**：监控图表禁止使用复杂渐变填充，避免干扰趋势判断。
    
*   **禁止状态模糊**：禁止用纯灰色表示非禁用状态。若任务挂起，需用明确的橙色标识。
    
*   **禁止截断报错**：在日志面板中，禁止直接截断关键错误栈，必须提供展开查看逻辑。

## 8. Responsive

针对 MLOps 及监控系统的多设备兼容特性（基于 `tailwind.config.js` 断点定义）：

| Breakpoint | Width Range | Layout Behavior |
| --- | --- | --- |
| 超大屏终端 | `>= 1470px` (`2xl`) | 完整布局铺开，拓宽主数据表格，展示所有数据列 |
| 桌面端 (标准) | `1024px - 1280px` (`lg` / `xl`) | 左侧边栏保持展开，表格操作列可能需要固定在右侧，内部数据横向滚动。 |
| 平板/小屏桌面 | `768px - 1023px` (`md`) | 左侧菜单栏（`LeftArea`）默认收缩为图标模式以让渡横向空间，数据表格开启自适应折叠。 |

**额外适配要求**：

*   **图表自适应**：系统集成的 `@antv/g2plot` 和 `echarts` 在容器调整（如展开收起侧边栏）时，必须监听 resize 事件重新渲染。
    
*   **监控大屏场景**：对于全屏模式的监控数据卡片，支持基于 VW/VH 的响应式流体缩放。

## 9. Agent prompt guide

在利用 AI 或 Cursor 进行本调度系统的页面和组件生成时，请将以下规则注入 Prompt：

*   **核心色值简表**：`bg=#f5f5f5, surface=#ffffff, primary=var(--color-primary), text=rgba(0,0,0,0.85), border=rgba(0,0,0,0.15)`
    

*   **场景提示词参考**：
    

*   **监控大屏**：构建监控面板。要求：深色/浅色高对比，4px 紧凑间距，曲线图对齐 X 轴，主色高亮关键利用率。
    
*   **任务表单**：创建 MLOps 任务表单。要求：`.form-item-horizontal` 水平布局，右侧 1/4 区域实时预览配置 JSON，圆角 6px。
    
*   **日志诊断**：生成 Terminal 日志组件。要求：等宽字体，背景 `#ffffff`，支持关键词高亮，内嵌 `CopyButton`。
